# T1: 多模态大模型技术架构演进综述

> **来源说明**：本文基于 researcher 训练知识（截至 2025 年初）整理，受限于实时搜索不可用，部分 2025-2026 年最新进展可能未覆盖。建议 writer 后续补充最新数据。

## 1. 概述

多模态大模型（Multimodal Large Language Models, MLLMs）是指能够同时处理和理解文本、图像、音频、视频等多种模态输入与输出的大型模型。其技术架构在过去数年间经历了从"拼接"到"统一"的深刻变革。

## 2. 架构演进阶段

### 2.1 阶段一：双塔编码器（~2019-2021）

- **代表**：CLIP (OpenAI, 2021)、ALIGN (Google, 2021)
- **架构**：分别训练图像编码器（Vision Transformer/ResNet）和文本编码器（Transformer），通过对比学习在共享嵌入空间对齐
- **特点**：跨模态检索能力强，但生成能力弱；两个编码器独立推理
- **局限**：无法生成图像或进行复杂的多模态推理

### 2.2 阶段二：编码器-解码器拼接（~2022-2023）

- **代表**：Flamingo (DeepMind, 2022)、BLIP-2 (Salesforce, 2023)、LLaVA (2023)
- **架构**：冻结 LLM 作为核心，通过适配层（如 Q-Former、线性投影）连接视觉编码器
- **关键创新**：
  - **Q-Former** (BLIP-2)：用一组可学习的查询 token 桥接视觉编码器与 LLM，大幅减少训练参数
  - **视觉投影** (LLaVA)：将 CLIP 图像特征通过 MLP 投影到 LLM 词嵌入空间
- **特点**：利用已有 LLM 的强大推理能力，仅需少量可训练参数
- **局限**：视觉信息被压缩，可能丢失细粒度空间信息

### 2.3 阶段三：原生多模态统一架构（~2023-2025）

- **代表**：GPT-4V/GPT-4o (OpenAI, 2023-2024)、Gemini (Google, 2023-2024)、Pixtral (Mistral, 2024)、Janus (DeepSeek, 2024)
- **架构特征**：
  - 端到端训练，非拼接式
  - 统一的 token 序列处理文本、图像、音频
  - 图像/视频分块（patch tokenization）后与文本 token 交织
  - 部分采用 Mixture of Experts (MoE) 提升效率
- **关键进展**：
  - **GPT-4o**：原生多模态，统一的语音-视觉-文本端到端模型，延迟大幅降低
  - **Gemini 1.5/2.0**：支持 100 万+ token 长上下文，原生多模态理解
  - **Any-to-Any 生成**：从理解走向生成（图像生成、语音合成）

### 2.4 阶段四：高效多模态与专用架构（2025+）

- **趋势**：
  - **动态分辨率**：不再固定裁剪，支持任意分辨率/长宽比输入
  - **MoE 多模态**：不同模态激活不同专家子集，提升效率
  - **原生语音模型**：如 GPT-4o 的端到端语音交互
  - **视频原生理解**：长视频时序建模成为核心挑战
  - **Agent 化多模态**：多模态模型具备工具调用与环境交互能力

## 3. 关键技术组件

### 3.1 视觉编码
| 方法 | 代表 | 特点 |
|------|------|------|
| ViT + CLIP | LLaVA | 成熟稳定，分辨率受限 |
| 动态分辨率 ViT | Pixtral, InternVL | 适应任意尺寸 |
| 原生视觉分词器 | Chameleon (Meta) | 视觉也作为离散 token |

### 3.2 模态对齐
- **投影层**：MLP/线性映射（简单高效）
- **Q-Former/Bridge**：查询驱动的信息瓶颈
- **交叉注意力**：早期融合，计算量大
- **交错训练**：多模态数据自然交织（如 GPT-4o）

### 3.3 训练策略
- **两阶段训练**：先对齐预训练，再指令微调
- **三阶段训练**：增加多模态预训练中间阶段
- **直接偏好优化 (DPO)**：用于多模态对齐

## 4. 视频理解架构

- **时序建模**：3D ViT、时间注意力、帧采样策略
- **长视频**：稀疏采样 + 记忆模块，或利用长上下文能力
- **视频生成**：扩散模型（Sora, 2024）与自回归模型并行发展

## 5. 音频-语言架构

- **语音识别→理解**：Whisper → LLM pipeline（早期）
- **端到端语音**：GPT-4o 式原生语音处理，避免 ASR 信息损失
- **音频事件理解**：AudioSet 为代表的音频分类与 LLM 融合

## 6. 架构对比总结

| 维度 | 双塔 | 拼接式 | 原生统一 |
|------|------|--------|----------|
| 跨模态推理 | 弱 | 中 | 强 |
| 端到端训练 | ✗ | 部分 | ✓ |
| 任意模态输出 | ✗ | ✗ | ✓ |
| 训练成本 | 低 | 中 | 高 |
| 信息保真度 | 低 | 中 | 高 |
